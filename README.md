# BlinkIT Grocery Sales Analysis:

## Overview
This project analyzes BlinkIT grocery sales data using SQL to identify key sales drivers, outlet performance patterns, and product-level insights. The objective is to convert raw transactional data into meaningful business insights.

## Objectives
- Clean and prepare raw sales data
- Calculate key business metrics such as total sales, average sales, item count, and ratings
- Analyze sales performance by product, outlet, size, and location
- Perform advanced analysis using trends, rankings, and efficiency metrics

## Tools & Skills
- Database: MySQL
- Language: SQL
- Concepts Used:
  - Data Cleaning
  - Aggregations & Grouping
  - CASE WHEN logic
  - Subqueries
  - Window Functions

## Key Analyses
- Core KPIs: Total sales, Average sales, Number of items, Average ratings
- Product analysis by item type and fat content
- Outlet performance by size, location, and establishment year
- Percentage contribution and efficiency analysis
- Time-based trends and ranking analysis
- Outlet Efficiency Metrics
- Location x Product Cross Analysis


## Conclusion
- Sales are driven by a small set of high-performing products and outlet types.
- Outlet size, location, and establishment year significantly influence sales performance.
- Combining sales, ratings, and efficiency metrics supports data-driven business decisions.
